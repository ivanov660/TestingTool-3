  1. Описание команд для API Selenium 

В данном случае сервис выполняет роль посредника между менеджером и Selenium. Основная задача, которая возлагается на данное взаимодействие, это осуществление имитации действий пользователя с интерфейсом. Технические проверки, которые может выполнять Selenium остаются за рамками данного инструмента. 

Список доступных команд приведен ниже: 

  • Нажать – выполняет отправку команды нажатия кнопки в браузер для текущего элемента 

  • Двойной клик – выполняет отправку двойного щелчка в браузер для текущего элемента 

  • Фокус – устанавливает фокус на текущем элементе  

  • Установить текст – выполняет действие по установке текста. 

  • Отправить key – отправляет клавишу или комбинацию клавиш 

  • Перейти по адресу – выполняет передачу браузеру команды перехода по url адресу 

  • Установить неявное ожидание – выполняет установку неявного ожидания в секундах.  

  • Нажать правая кнопка – выполняет команду передачи 

  • Получить родителя – выполняет команду получения родителя текущего элемента. 

  • Очистить – очищает содержимое элементов редактирования элементов html «input», «textarea». 

  • Получить текст – получает текстовое содержимое элемента. 

  • Получить атрибут – получает значение атрибута элемента html. 

  • Ctrl+A –выполняет команду выделения содержимого в рамках текущего элемента. 

 

  1. Описание команд для API Automation UI 

Для части команд используется приставка «simulate» – она обозначает что используются дополнительные возможности операционной системы для выполнения описанного действия, когда элемент тестируемого клиента не поддерживает формат интерфейса «Microsoft Automation UI». Для решения обозначенной проблемы используются специальные команды операционной системы Microsoft Windows - «PostMessage» (асинхронная) или «SetMessage» (синхронная). Эти команды вызывают симуляцию действия для определенного выбранного окна (вы в скрипте прежде ее вызова должны найти основное окно или подчиненное тестируемого клиента), благодаря такому подходу появляется возможность использования множественного запуска сценариев тестирования в одном сеансе (однако это зависит от структуры скрипта).  

Список доступных команд приведен ниже: 

  • Нажать – выполняет попытку нажатия вызовом соответствующей функции элемента. Не все объекты поддерживают стандарт «Microsoft Automation UI», поэтому при отсутствии результата пробуйте использовать «Нажать (simulate)». 

  • Нажать (simulate) – выполняет симуляцию команды нажатия левой кнопки мышки в определенное текущее окно с использованием функции «PostMessage». 

  • Нажать % - выполняет нажатие кнопки в поле относительно левого верхнего и нижнего правого угла размеров элемента.  
    К примеру, у вас есть текстовый элемент, в котором есть кнопка выпадающего списка, относящегося к этому элементу и нет возможности на нее нажать, тогда прикинув в % отклонение, то вы сможете с позиционировать курсор мышки на этом поле. 

  • Нажать правая кнопка (simulate) – выполняет симуляцию команды нажатия правой кнопкой мышки в определенное текущее окно с использованием функции «PostMessage». Обычно эта команда требуется, когда необходимо вызвать контекстное меню элемента. 

  • Двойной клик – выполняет попытку двойного нажатия мышки. 

  • Двойной клик (simulate) – выполняет симуляцию команды двойного нажатия правой кнопкой мышки в определенное текущее окно с использованием функции «PostMessage». 

  • Фокус – устанавливает фокус на элемент. 

  • Установить текст – устанавливает текст в поле элемента, вызывая соответствующие команды поддерживаемых этот функционал элементов – «Edit Box», «Rich Text», «Document» и другие. 

  • Установить текст (simulate) – выполняет симуляцию команды отправки текста в определенное текущее окно. 

  • Отправить key – отправляет символ или комбинацию клавиш в текущий элемент. 

  • Отправить key (simulate) – отправляет символ или комбинацию клавиш в текущий элемент определенного текущего окна. 

  • Переключить флаг – выполняет команду переключения флага элемента «Check Box». 

  • Развернуть или свернуть – выполняется попытка отправки для элемента команды развернуть, если свернут или свернуть, если развернут. 

  • Установить неявное ожидание – определяет глобальное в секундах ожидание поиска элементов в течении, которого система не будет выдавать ошибку. По умолчанию это значение равно 0 секунд. Для поиска каждого элемента можно установить свое время (перетереть глобальное).  
    Если объект будет найден менее чем за указанный интервал, то система продолжит выполнение скрипта иначе будет вызвано исключение о ненайденном элементе. 

  • Получить родителя – получает родительский элемент в текущем контексте и устанавливает на него фокус, дальнейшие подчиненные действия будут проводиться с родителем. 

  • Получить текст – пытается получить текст текущего элемента. Из редактируемых полей будет получен текущий текст, для других будет попытка получения заголовка и значения надписи. 

  • Получить заголовок окна (simulate) – получает заголовок окна с использованием дополнительных функций. 

  • Закрыть окно – вызывает команду закрытия текущего окна, не применима для любых других элементов. 

  • Текст из буфера обмена – получает текст в переменную.  
    Происходит работа с буфером обмена, он один на весь текущий сеанс, поэтому если выполняете несколько скриптов одновременно с использованием буфера обмена, учитывайте это ограничения для предотвращения коллизий.  

  • Установить в буфер обмена – передает текст в буфер обмена.  
    Происходит работа с буфером обмена, он один на весь текущий сеанс, поэтому если выполняете несколько скриптов одновременно с использованием буфера обмена, учитывайте это ограничения для предотвращения коллизий. 

  • Получить текст под фокусом – пытается получить текст для элемента, отмеченного фокусом. Выполняет команду выделения, копирования в буфер обмена и получения из буфера обмена.  
    Происходит работа с буфером обмена, он один на весь текущий сеанс, поэтому если выполняете несколько скриптов одновременно с использованием буфера обмена, учитывайте это ограничения для предотвращения коллизий. 

  • Ожидать отклика – пытается получить информацию о том, что приложение запустилось, открылось основное окно и тестируемый клиент готов к обработке команд тестирования. Однако из-за определенных технических проблем, вызываемых в ходе внутренней обработки в приложении, может потребоваться еще дополнительная задержка, поэтому дополнительно используйте задержку таймера или иные способы определения, что приложение готово.
