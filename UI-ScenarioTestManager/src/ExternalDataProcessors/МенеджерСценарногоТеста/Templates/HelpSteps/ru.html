1. Описание типов шагов редактора сценария 

Сценарий тестирования состоит из шагов, имитирующих действие пользователя с графическим интерфейсом (нажать кнопку, закрыть окно и др.). Шаг структурно является минимальным элементом сценария. Запись и редактирование сценария выполняются в любой базе (оптимально в эталонной базе). 

В процессе редактирования каждого шага вы легко можете сменить тип этого шага. 

Данного набора действий оптимально и достаточно для сценариев любой сложности. 

  1. Настройки общие для всех типов шагов 

Настройка шагов выполняется при помощи свойства панели «свойств шага». Состав свойств панели «свойства шага» зависит от типа настраиваемого шага. Некоторые настраиваемые параметры являются общими для всех типов шагов: 

  • Номер шага – номер шага. Это поле следует рассматривать только как информационное, так как номера шагов при добавлении и удалении шагов в сценарий могут пересчитываться. 

  • Активность – активность текущего шага. Шаги, у которых флажок Активность снят, не выполняются. 

  • Наименование – наименование шага для отображения в дереве сценария. 

  • Действие — тип выполняемого действия, определяет функциональность шага. Типы действий будут разъясняться далее для каждого шага. 

  • Продолжить при ошибке — позволяет продолжить выполнение шага при возникновении критической ошибки (fail). Эта ошибка в отчете будет интерпретироваться как error (ошибка), а не fail (падение). Установка данного признака допускается, если сценарный тест может продолжить свое выполнение. 

  • Комментарий – комментарий к шагу. 

  • Описание шага – описание шага. В зависимости от положения флажка не выводить авто описание отображается либо только пользовательское описание шага, либо пользовательское описание и описание, сгенерированное автоматически. 

  • API – тип используемого программно-аппаратного интерфейса для выполнения текущего шага. Рекомендуется разделять интерфейсы на некоторые блоки шагов. Переключение/пере подключение выполняется шагом действием «Подключить тестируемое приложение».  
    Могут иметь следующие значения: 

  • Пустое значение – считается платформа 1С; 

  • Application 1C – платформа 1С; 

  • Selenium – Фреймворк Selenium; 

  • Automation UI – Microsoft Automation UI. 

  1. Настройка шага «Комментарий» 

Шаг комментарий предназначен для добавления пустого шага (не выполняет никаких действий). Его можно использовать как логическую группировку шагов, разделитель между группами шагов или как некоторое пояснение в рамках сценария. 

К примеру, если вы записываете тест в браузере для приложения 1С Enterprise, то в этом случае не существует общедоступных понятий найти окно или найти форму как при использовании API 1C Автоматизация. В этом случае вы можете создать шаг комментарий «Найти окно», а внутри него выполнить все необходимые действия по поиску и выполнению дополнительной группировки. Это позволит более четко выполнить структуризацию сценария и внесет больше ясности пользователям. Параметры приведены ниже: 

  • Действие. Значение = «Комментарий» 

  1. Настройка шага «Готовый блок шагов» 

Шаг «готовый блок шагов» предназначен для добавления в текущий сценарий готовых наборов шагов, которые были записаны ранее и сохранены в мини сценарий для дальнейшего повторного использования. Обычно данные блоки хранятся в отдельном каталоге, называющемся «библиотекой».  

Данный функционал позволяет значительно упростить создание сценариев, за счет повторного использования мини сценариев из библиотеки. Упрощает исправление и поддержку итоговых сценариев, за счет исправления одного скрипта в библиотеки, а не всех сценариев подряд. 

К примеру, вам требуется выполнить поиск документа в динамическом списке или журнале программы 1С. Соответственно, если вы напишите один раз скрипт поиска с входными параметрами (номер документа или наименование справочника), тогда вы сможете его вызвать в других сценариях, где понадобится поиск в подобных списках платформы 1С и не только. 

Параметры: 

  • Действие. Значение = «Готовый блок шагов» 

  • Путь к библиотеке – путь к библиотеке сценариев, по умолчанию определяется в настройках менеджера. Подставляется автоматически при добавлении шага или загрузке скрипта сценария. Хранится относительный путь, если путь совпадает с локальным каталогом.  
    Не рекомендуется вручную менять и писать локальный путь на текущем компьютере – это может привести к неработоспособности скрипта на других компьютерах. 

  1. Настройка шага «Команда» 

Шаг позволяет выполнить определенное действие для текущего элемента или окна тестируемого клиента. Команды имеют определенную специфику и наборы свойств, которые могут быть уникальными. Для каждого типа элемента может быть свой уникальный набор команд. Список команд и перечень используемых свойств приведен ниже в разделе «Описание команд». 

Параметры: 

  • Действие. Значение = «Команда» 

  • Имя команды – имя выбранной команды, определяется типом объекта применения. 

  • Другие свойства определяются типом команды и приведены в разделе «Описание команд».   

 

  1. Настройка шага «Таймер» 

 

Шаг позволяет в логику скрипта добавлять задержку на некоторый интервал времени в течение которого не будет выполняться никаких действий. Рекомендуем быть осторожным с этим параметром, т.к. он будет увеличивать на постоянное значение время выполнения сценария. Не стоит устанавливать этот шаг для ожидания появления элемента в иерархии тестируемого клиента, вместо этого используйте свойство интервал у объектов поиска или шаг с типом «элемент существует». 

Параметры приведены ниже: 

  • Действие. Значение = «Таймер» 

  • Интервал – время в секундах, на которое будет прервано выполнение скрипта (явное ожидание). 

 

  1. Настройка шага «Найти основное окно» 

Доступно только для API 1С и Automation UI  

Шаг позволяет найти основное (родительское/главное) окно тестируемого клиента. После его успешного нахождения автоматически указатель текущее окно будет ссылаться на него, иначе будет выдано сообщение об ошибке. 

Параметры приведены ниже: 

  • Действие. Значение = «НайтиОсновноеОкно» 

  • Заголовок – наименование заголовка. Поддерживается добавление операторов подстановки параметров &Параметр. К примеру, «Заказ клиента &НомерДокумента». 
    Для платформы 1С дополнительно поддерживаются теги «*» - любые символы и любое количество и «?» - любой одиночный символ. 

  • Интервал – время в секундах, на которое будет отведено время поиска. Если окно будет найдено ранее, то скрипт немедленно продолжит работу.

  1. Настройка шага «Подключить тестируемое приложение» 

Шаг позволяет выполнить подключение к тестируемому клиенту. Этот шаг обязателен в начале готового скрипта, иначе не будет проведен поиск и подключение к клиенту. В случае успешного подключения скрипт продолжит выполнение, а в случае ошибки будет выдано информационное сообщение об ошибке и выполнение будет прервано с ошибкой «fail» 

Параметры приведены ниже: 

  • Действие. Значение = «ПодключитьТестируемоеПриложение»  

  • Интервал – время ожидания подключения в секундах. 

  • Свойства подключения – дополнительные параметры подключения. Доступны три режима: 

  • Пустое значение – используются глобальные настройки подключения 

  • Значение параметром – будет использоваться данные из параметра, который должен будет описан в таблице «параметры/переменные сценария». К примеру, может быть задано следующее значение - «&МенеджерERP». Если в таблице параметров не будет найден параметр с таким именем «МенеджерERP» (без «&»), то будут использоваться данные по умолчанию (эквивалентно «Пустое значение»). 

  • Задано значение настройки – параметры заданы через форму редактирования, см. рисунок ниже.

  1. Настройка шага «Код клиент» 

Шаг позволяет выполнить произвольный код на языке 1С Предприятие. Этот шаг следует использовать, когда не хватает типового функционала. Код выполняться будет в рамках клиента. 

  1. Настройка шага «Код сервер» 

Шаг позволяет выполнить произвольный код на языке 1С Предприятие. Этот шаг следует использовать, когда не хватает типового функционала. Код выполняться будет в рамках сервера.  

  1. Настройка шага «Получить представление данных» 

Шаг позволяет получить текстовое представление текущего элемента и поместить это значение в параметр для дальнейшего использования в последующих алгоритмах. 

  1. Настройка шага «Сравнить данные»  

Шаг позволяет выполнить сравнение данных - assert. В случае несоответствия ожиданию будет вызвано исключение и прервано выполнение ошибки (fail). Если для шага поставить значение поля «продолжать в случае ошибки» в истина, то работа команды будет эквивалентна операции verify и в случае ошибки значение будет равно типу error и выполнение будет продолжено, т.е. тест будет «битым». 

  1. Настройка шага «Пауза» 

Шаг позволяет приостановить выполнение скрипта на этом шаг. Далее система будет ожидать действия оператора и имеется возможность восстановить выполнение сценария. Данный шаг рекомендуется использовать только для режима отладки и разработки скрипта. 

 

  1. Настройка шага «Стоп» 

Шаг позволяет завершить выполнение скрипта на этом шаге и происходит сброс в начало. Далее система будет ожидать действия оператора. Данный шаг рекомендуется использовать только для режима отладки и разработки скрипта. 

  1. Настройка шага «Условие» 

Шаг типа «условие» выполняет организацию ветвления - позволяет выполнить проверку на равенство двух параметров и в случае выполнения условия (истина), продолжить выполнение подчиненных шагов данного элемента узла, иначе выполняется переход на следующий шаг в рамках текущего уровня. На рисунке приведены два примера демонстрирующие работу алгоритма. Первое условие выполняется и выполняются все шаги внутри условия, а результат проверки второго равен ложь – и все шаги пропускаются. 

 

Параметры: 

  • Действие – значение равно «Условие» 

  • Имя параметра – имя первого параметра из таблицы «параметры/переменные сценария», откуда берется значение 

  • Имя параметра 2 – имя второго параметра из таблицы «параметры/переменные сценария», куда передается значение (если не заполнен, то происходит сравнение с значением параметра) 

  • Значение параметра – константа, по которой происходит проверка. 

  • Условие сравнения – условие в рамках которого выполняется проверка. Доступные комбинации: 

  • Равно – проверка на равенство 

  • Не равно – проверка на неравенство 

  • Заполнено – проверка на заполнено (есть какое-либо значение) 

  • Не заполнено – проверка не заполнено (пустое значение) 

  • Содержит – на то, что в тексте встречается значение 

  • Не содержит – на то, что в тексте не встречается значение 

  • RegExp – соответствует шаблону 

  • Не RegExp – не соответствует шаблону 

 

 

  1. Настройка шага «Параметр1-> Параметр2» 

Шаг позволяет передать значение одного параметра в другой. Данный шаг может использоваться в логике сценария, когда требуется передача данных между параметрами. К примеру:  

У вас в библиотеке есть скрипт создающий документ заказ клиента на основании коммерческого предложения. Входной параметр для этого скрипта «Номер коммерческого предложения», а в текущем разрабатываемом скрипте вы создаете заказы на основании двух разных коммерческих предложениях с параметрами «Номер КП1» и «Номер КП2». Поэтому чтобы выполнить независимый скрипт вам требуется сначала передать из параметра «Номер КП1» в параметр «Номер коммерческого предложения», затем выполнить скрипт из библиотеки, а потом повторить эту операцию со вторым коммерческим предложением. 

Параметры: 

  • Действие – значение равно «ИзПараметра1ВПараметр2» 

  • Имя параметра – имя первого параметра из таблицы «параметры/переменные сценария», откуда берется значение 

  • Имя параметра 2 – имя второго параметра из таблицы «параметры/переменные сценария», куда передается значение 

 

  1. Настройка шага «Элемент существует» 

Шаг предназначен для быстрой проверки наличия элемента в иерархии тестируемого клиента. Данный шаг применим ко всем типам объектов. 

Параметры: 

  • Действие – значение равно «ПроверкаНаличияЭлемента» 

  • Тип объекта – тип объекта поиска (может не заполняться) 

  • Имя объекта – имя объекта поиска (может не заполняться) 

  • Заголовок объекта – заголовок объекта поиска (может не заполняться) 

  • Имя параметра – в эту переменную будет записываться результат поиска, это значение можно использовать далее в шаге с типом «Условие» 

  • Интервал – время в секундах на протяжении которого происходит ожидание появления элемента. Если элемент появился ранее значения интервала, то шаг будет продолжен сразу. 

  • Варианты поиска – использовать или нет варианты поиска (по умолчанию не использовать) 

 

  1. Настройка шага «Вызвать исключение» 

Шаг предназначен для вызова исключения и прекращения работы скрипта с формированием события падения (fail), в частности для ситуаций, когда по некоторому логическому условию продолжение выполнения скрипта будет бессмысленным. Данный подход позволит серьезно повысить понятность причин падения сценария тестирования. К примеру: 

  • Вы проверяете при старте системы появления окна авторизации при запуске предприятия, если оно появилось, то авторизация через команду запуска не сработала и продолжение бессмысленно.  

  • Или вы после нажатия кнопки «провести и закрыть» документа (ожидаемое поведение проведение документа и закрытие окна) проверяете наличие открытого окна документа в этом случае продолжение выполнения сценария бесполезно. Однако, падение может произойти значительно дальше ошибочной точки и это затруднит понятие причин, приводящих к ошибке. 

Параметры: 

  • Действие – значение равно «ВызватьИсключение» 

  • Вводимый текст – определяет сообщение формируемой ошибки, может поддерживать параметры в строке «&Параметр». Пример: «Документ &НомерДокумента не провелся, а должен был!» 

  1. Настройка шага «Сделать Screenshot» 

Шаг предназначен для создания снимка экрана для текущего состояния системы, вне зависимости от установленного значения генерации шагов в настройках менеджера. Для выполнения этой операции обязательно наличие запущенного внешнего сервиса «WinAutomationUI» и выполненных настроек в разделе «Внешние инструменты». 

Данный шаг может использоваться для оптимизации создания количества снимков экранов. 

Параметры: 

  • Действие – значение равно «СделатьСнимокОкна» 

  1. Настройка шага «Память» 

Шаг предназначен для сохранения данных параметров во внешних файлах или базе данных.  

Параметры: 

  • Действие – значение равно «Память»
