Внешний сервис WinAutoamtionUI

  1. Назначение 

Предназначена для выполнения действий взаимодействия с API Selenium, Microsoft Automation UI, выполнению вспомогательных действий (сохранение скриншотов, запуск приложений и др). 

Скомпилированный вариант приложения можно скачать и посмотреть на следующей странице: http://testingtool.ru/external-user-interface-testing-tool-winautomationui/

  1. Описание API 

 

Взаимодействие осуществляется через REST-интерфейс. Набор команд в строке подключения: 

http://localhost:8088/rest.html?&Operation=status_session&answer_format=json&session_id=42 

 

Свойства общие 

 

  • Operation – операция. Определяет текущую команду. Доступны: 

  • status_session 

  • new_predefined_session 

  • play_step – проиграть шаг/выполнить действие автоматизации 

  • answer_format – формат ответа. Доступны json (оптимальный), xml, text. 

  • session_id – идентификатор сессии. Для приложений Windows рекомендуем отправлять PID, для Web номер порта тестирования 

  • api – апи с которым работает. Доступны два значения «Selenium» или «Automation UI» 

  • action – команы работы с тестируемым клиентом 

  • «command» - команда 

  • «find main window» – найти главное/основное окно 

  • «find window» – найти окно 

  • «find element» - найти элемент (среди всех элементов подчиненного окну) 

  • «find element child» - найти элемент подчиненный (один уровень вниз) 

  • process_id – номер процесса тестируемого клиента. Актуально для api= «Automation UI» 

 

Свойства элементов и команд 

 

  • element_name – имя элемента. Может быть не задано. Если задано, то поиск осуществляется по И. Может содержать один элемент *. В этом случае будет искаться вхождение фразы. К примеру, “Управление предприятием версия 2.4*”. Будут найдены элементы “Управление предприятием версия 2.4 – Пользователь Петрова А.А.”. 

  • interval – максимальное время поиска объекта/элемента. Элемент должен быть найден ранее чем истечет время интервала. Если находится быстрее, то сразу происходит завершение шага успешное. 

  • element_class_name – имя класса. Может быть не задано. Если задано, то поиск осуществляется по И. Может содержать один элемент *. В этом случае будет искаться вхождение фразы. К примеру, “V1CMain*”. Будут найдены элементы “V1CMainForm”, “Red V1CMainForm”. 

  • element_type – тип элемента 

  • «Automation UI» 

  • «» - пустой, означает любой элемент 

  • «window» - окно 

  • «button» - кнопка 

  • «checkbox» - флажок 

  • «edit» - элемент, редактируемый текст 

  • «tabitem» - элемент вкладки 

  • «text» - текстовый элемент 

  • «radiobutton» - радио кнопка 

  • «pane» - панель 

  • «menuitem» - элемент меню 

  • «custom» - элемент, имеющий тип разработчика 

  • «Selenium». Типы смотрите в описании HTML стандарта. 

  • «» - пустой, означает любой элемент 

  • «div» - тег слоя 

  • «a» - тег ссылки 

  • «input» - тег элемента управления 

  • «label» - надпись 

  • «span» - произвольный элемент оформления 

  • «textarea» - тег области редактирования 

  • «img» - тег картинки 

  • «li» - тег нумератора 

  • «td» - тег таблицы 

  • element_variable_name – имя переменной элемента 

  • element_parent_variable_name – имя элемента переменной родителя 

  • element_xPath – путь xPath 

  • element_text – текст для команд вставки текста 

  • command – имя команды 

  • «Selenium» 

  • «find element» - найти элемент 

  • «get text» - получить текст 

  • «get attribute» - получить текст атрибута 

  • «click» - команда нажать 

  • «dblclick» - двойной клик 

  • «focus» - установить фокус на элемент 

  • «text» - установить текст 

  • «send key» - отправить код клавиши. Отправить комбинации. Пример «Ctrl+A», «Ctrl+C», «Ctrl+V» 

  • «navigate» - перейти по адресу страница 

  • «implicit wait» - установить неявное ожидание по умолчанию для всех действий 

  • «rclick» - нажать правой кнопкой мыши 

  • «parent» - установить текущим элементом родителя выбранного элемента 

  • «clear» - очистить 

  • «ctrl plus a» - выделить все 
     

  •  «Automation UI». Для команд приставка simulate – выполняет низкоуровневую отправку команд через потоки сообщений окна. 

  • «find main window» - найти основное окно (главное) 

  • «find window» - найти подчиненное окно 

  • «find element» - найти элемент 

  • «find element child»- найти подчинённый элемент на 1 уровень вниз 

  • «click» - нажать 

  • «simulate click» - нажать с помощью средств отправляет низкоуровневое сообщение нажатия, когда не работает типовоя команда «click» 

  • «click%» - нажать по позиции в процентах внутри контейнера контролла элемента, может перемещать мышь в позицию элемента 

  • «simulate click%» - нажать в позицию в процентах размера контрола элемента, через низкоуровневые команды 

  • «simulate rclick» - имитирует нажатие правой клавиши мышки в текущее окно элемента и на текущий выбранный объект 

  • «dblclick» - отправляет двойной клик 

  • «simulate dblclick» - отпавляет низкоуровневую команду нажатия 

  • «focus» - устанавливает фокус на элемент, делает его активным 

  • «text» - установка текста в текущий элемент. Только для поддерживающих элементов типа textbox, input и др. 

  • «simulate text» - устанавливает текст на низком уровне 

  • «send key» - отправляет клавишу или сочетания клавиш. Пример: «Ctrl+A», «Shift+Alt+F» 

  • «simulate send key» - отправляет клавишу или сочетания клавиш. 

  • «toggle» - переключить флаг элемента checkbox 

  • «expand or collapse» - схлопывает или раскрывает узел дерева 

  • «implicit wait» - установить неявное ожидание для всех контроллов по умолчанию 

  • «parent» - установить текущем родителя выбранного элемента 

  • «get text» - получить текст элемента или заголовок (не текстовых типа label) 

  • «simulate get window title» - низкоуровневая команда получения заголовка окна 

  • «close» - закрыть окно 

  • «clipboard get text» - получить текст из буфера обмена 

  • «clipboard set text» - получить текст из буфера обмена 

  • «simulate get focused text» - получить текст элемента под фокусом (через команду копирования в буфер и получения из буфера), к примеру, можно получить текст выделенной ячейки табличного документа. 

  • «wait for input» - ожидает доступности/появления окна (но не всегда окно становится доступным для работы, рекомендуется еще добавить таймер) 

 

  • element_id – идентификатор элемента 

  • clickX – позиция по X внутри размеров контролла 

  • clickY – позиция по y внутри размеров контролла 

  • element_title – заголовок элемента 

  • element_attribute – атрибут элемента для «Selenium» обращение к значениям элементов свойства тега класс, стиль, заголовок и т.п. Пример, для тега <div class=”orange-class”> обращение к атрибуту «class» будет получено значение «orange-class» командой «get attribute»
